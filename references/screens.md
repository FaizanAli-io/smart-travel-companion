This design showcase for the **Smart Travel Companion** app outlines a modern, clean, and highly functional mobile UI. The aesthetic relies on a deep purple primary color, rounded corners, and high-quality imagery.

Here is a detailed breakdown and description for every screen required based on the provided design:

---

## 1. Home Screen

The central hub for discovery. It features a top bar with a hamburger menu, title ("Explore Places"), and a notification bell.

- **Search & Filters:** A prominent search bar is followed by horizontal filter chips (All, Favorites, Recent).
- **Content Feed:** A vertical list of destination cards. Each card includes a large hero image, destination name (e.g., Lake Tekapo), location, and a heart icon for favoriting.
- **Navigation:** A persistent bottom navigation bar with five icons (Home, Map, Add/Floating Action Button, Favorites, Profile).

## 2. Detail Screen

The deep-dive view for a specific location.

- **Visuals:** A large, high-resolution header image with a back button and a favorite toggle overlaid at the top.
- **Information:** Displays the location name, region, and a text description ("About the place").
- **Weather Widget:** A dedicated "Current Weather" section showing temperature (16°C), conditions (Clear Sky), wind speed, humidity, and "feels like" metrics.
- **Action:** A primary button at the bottom labeled "View on Map."

## 3. Search & Filter Screen

The interface for narrowing down results.

- **Input:** An active search field (showing "Lake").
- **Refinement:** A dropdown for "Sort By" (Recommended) and toggle buttons for "Show" (All vs. Favorites).
- **Region Selection:** A dropdown menu to filter by specific geographic regions.
- **Results:** A scrollable list of mini-cards showing the filtered results (Lake Tekapo, Lake Bled, etc.).

## 4. Empty State

Displayed when a search or filter yields no results.

- **Illustration:** A stylized purple graphic of a magnifying glass over a folder.
- **Messaging:** "No places found" with sub-text advising the user to adjust their filters.
- **Call to Action:** A "Clear Filters" button to reset the view.

## 5. Favorites Screen

The user’s curated list of saved destinations.

- **Layout:** Similar to the home screen but filtered specifically for user-saved items.
- **List Items:** Compact cards showing the image, title, and a filled red heart icon indicating the item is saved.

## 6. Offline State

The UI fallback for when internet connectivity is lost.

- **Illustration:** A graphic of a mobile phone with a "no signal" slash.
- **Messaging:** "You’re offline" with a note that the app is showing cached places.
- **Call to Action:** A "Retry" button to attempt to reconnect.

## 7. Side Drawer (Navigation Menu)

A slide-out menu for account management and app settings.

- **Header:** User profile section with an avatar, name (Aarav Mehta), and email address.
- **Links:** A list of vertical menu items: Home, Map, Favorites, Downloaded, Settings, Help & Support, and About Us.
- **Toggle:** A "Dark Mode" switch at the bottom for quick theme switching.

---

## 8. Light & Dark Theme Comparison

This view demonstrates the app's versatility in different lighting conditions.

- **Light Theme:** Uses a white background with dark text for high legibility.
- **Dark Theme:** Uses a deep charcoal/navy background with white text and vibrant purple accents to reduce eye strain.

---

## UI Component Library (Style Guide)

To ensure consistency across these screens, the following components are defined:

- **Buttons:** Primary (solid purple) and Outline styles.
- **Chips:** Rounded pills for category selection.
- **Icons:** Thin-stroke line icons for navigation and actions.
- **Cards:** Standardized containers for destinations and weather data.
- **Color Palette:** A range from deep purple (`#5C63FF`) to neutral grays and alert colors (red/green).
- **Typography:** Uses the **Poppins** font family in various weights (Bold to Regular).
